# 「満腹待ち」スマホアプリ 要件確認書・追加仕様案

作成日: 2026-06-26

## 1. このドキュメントの目的

`spec.md` の内容をもとに、スマホアプリとして開発を始める前に決めるべき事項を整理する。

特に、追加要望である「クリプトニンジャのキャラが応援してくれる感じで、キャラも使いたい」を含めて、MVPに入れる範囲と後回しにする範囲を明確にする。

この `spec2.md` では、Kuroさんに決めてほしい項目を `要確認` としてまとめる。

## 2. 現時点で固めてよさそうな基本方針

## 2.1 アプリの目的

食べる量を無理に減らす前に、食べる速度を整える。

満腹感が追いつくまで、アプリとキャラクターが一緒に待ってくれる。

## 2.2 アプリ名

仮称: 満腹待ち

## 2.3 MVPの中心体験

最初のMVPでは、次の流れに絞る。

1. 食事前に今日の作戦を1つ選ぶ
2. 食事タイマーを開始する
3. 食事中にやさしいリマインドを受ける
4. 途中で満腹度を確認する
5. 食後に簡単に振り返る
6. 過去の記録を見る

## 2.4 MVPで扱う課題

- 早食い
- あまり噛まない
- 満腹感に気づく前に食べすぎる
- 自炊時の作りすぎ
- 盛りすぎ
- ながら食べ
- お酒による食べすぎの記録

ただし、飲酒日専用モードはMVPでは実装しない。

## 2.5 MVPでやらないこと

- カロリー計算
- 食品データベース
- 写真解析
- AIチャット
- 体重管理
- SNS機能
- 課金
- ログイン
- バックエンド
- 医療的な診断や断定
- 飲酒日専用モード
- PWA化
- プッシュ通知

## 3. スマホアプリとしての前提

`spec.md` ではWebアプリ想定も含まれていたが、今回の希望はスマホアプリ。

MVPの実装方針は、以下のどちらかを選ぶ必要がある。

## 要確認 A: 最初の実装形態

### A案: Webアプリをスマホ最適化して作る

特徴:

- 開発が最も速い
- URLで確認できる
- App Store申請が不要
- まず使い勝手を検証しやすい
- 通知、バイブ、バックグラウンド動作には制限がある

向いている場合:

- まず自分で使えるMVPを早く作りたい
- アプリストア公開より検証を優先したい

### B案: React Native / Expoでスマホアプリとして作る

特徴:

- iPhone / Androidアプリに近い体験にできる
- バイブや通知などが扱いやすい
- 将来ストア公開しやすい
- Webアプリより初期構築と検証に少し手間がかかる

向いている場合:

- 最初からスマホアプリとして育てたい
- 通知やキャラ演出をしっかり入れたい

### C案: まずWebアプリ、後でスマホアプリ化

特徴:

- MVP検証が速い
- 仕様が固まってからReact Native / Expoへ移行できる
- 二度作る可能性がある

向いている場合:

- 最初はスピード重視
- 使ってから本格アプリ化したい

推奨: C案。まずWebアプリでMVPを作り、体験が固まったらExpo化する。

Kuroさんの決定:

- [ ] A案: Webアプリ
- [ ] B案: React Native / Expo
- [これでお願いします] C案: まずWeb、後でスマホアプリ化

## 4. クリプトニンジャのキャラクター活用方針

## 4.1 追加コンセプト

食事中に、クリプトニンジャのキャラが「我慢しろ」と言うのではなく、やさしく応援してくれる。

キャラの役割は、ダイエットの監視役ではなく「伴走役」。

例:

- 急がなくて大丈夫
- ここで一度、箸を置こう
- 気づけた時点で前進
- 7割で止まれたら勝ち
- 満腹感が追いつくまで一緒に待とう

## 4.2 キャラの使いどころ

MVPでは、キャラを以下の場所に使う。

- トップ画面
- 作戦選択画面
- 食事タイマー画面
- 満腹度チェック
- 食後の振り返り
- 記録画面の達成表示

## 4.3 MVPでのキャラ演出レベル

実装負荷を抑えるため、MVPでは以下が現実的。

- キャラ画像は静止画
- 表情差分があれば、通常・応援・成功・気づきの4種類程度
- セリフは画面状態に応じて切り替える
- アニメーションは軽いフェードやバウンス程度

## 4.4 キャラを使う上での注意

クリプトニンジャのキャラクター画像・名称・二次利用条件は、実装前に利用許諾・ガイドライン確認が必要。

特に確認すべきこと:

- 商用利用してよいか
- アプリ内利用してよいか
- キャラ画像を改変してよいか
- クレジット表記が必要か
- NFTや公式素材との扱いの違い
- ストア公開時に問題がないか

この確認が終わるまでは、開発上は仮画像またはプレースホルダーで進めるのが安全。

## 5. 画面仕様案

## 5.1 トップ画面

目的:

食事開始まで迷わず進める。

表示:

- アプリ名
- 短いコンセプト文
- 応援キャラ
- 食事を始めるボタン
- 記録を見るボタン

文言案:

満腹待ち

食べる量を減らす前に、食べる速度を整えよう。

キャラセリフ:

今日も、満腹感が追いつくまで一緒に待とう。

## 5.2 作戦選択画面

目的:

今日の食事で意識することを1つだけ選ぶ。

選択肢:

- 15分以上かけて食べる
- ひと口ごとに箸を置く
- 明日の分を先に分ける
- おかわりしない
- 7割で一度止まる
- ながら食べをしない
- 水を飲んでから食べる

キャラセリフ:

今日の勝ち条件は1つで十分。小さく勝とう。

## 5.3 食事タイマー画面

目的:

食事のペースをゆっくりにする。

表示:

- 経過時間
- 目標までの残り時間
- 今日の作戦
- 応援キャラ
- リマインドメッセージ
- 満腹度チェックボタン
- 食事終了ボタン

基本仕様:

- 目標時間は15分
- 1分ごとにリマインド文言を切り替える
- 5分、10分で満腹度チェックを促す
- 15分で目標達成表示
- 15分後もタイマーは続ける

## 5.4 満腹度チェック

選択肢:

- 3割
- 5割
- 7割
- もう十分
- 食べすぎかも

キャラ応答例:

- 3割: まだ余裕あり。急がず進もう。
- 5割: いいペース。ここからゆっくりが大事。
- 7割: 一度、箸を置いて様子を見よう。
- もう十分: ここで終われたら大成功。
- 食べすぎかも: 気づけた時点で前進。ここから整えよう。

## 5.5 食後振り返り画面

記録項目:

- 今日の食べ方
- 食べすぎ原因
- 今日の作戦は守れたか
- メモ

今日の食べ方:

- ちょうどよかった
- 少し食べすぎた
- かなり食べすぎた

食べすぎ原因:

- 早食い
- 作りすぎ
- 盛りすぎ
- お酒
- ながら食べ
- ストレス
- もったいなくて食べた
- 空腹すぎた
- その他

キャラセリフ:

記録できたら十分。次の一食に活かそう。

## 5.6 記録画面

表示:

- 総食事記録数
- 15分以上食べられた回数
- 作戦を守れた回数
- 直近7日間の記録数
- よく出る食べすぎ原因トップ3
- 食事記録一覧

キャラセリフ:

できた日も、崩れた日も、全部ヒントになる。

## 6. データ仕様案

MVPではローカル保存のみ。

Webアプリなら `localStorage`。

React Native / Expoなら `AsyncStorage`。

保存するデータ:

```ts
type MealSession = {
  id: string;
  startedAt: string;
  endedAt: string | null;
  durationSeconds: number;
  targetDurationSeconds: number;
  selectedStrategy: MealStrategy;
  fullnessChecks: FullnessCheck[];
  result: MealResult | null;
  overeatingCauses: OvereatingCause[];
  strategyResult: StrategyResult | null;
  memo: string;
  supporterCharacterId: string;
};
```

```ts
type MealStrategy =
  | "eat_over_15_minutes"
  | "put_down_chopsticks"
  | "save_portion_for_tomorrow"
  | "no_second_serving"
  | "pause_at_70_percent"
  | "no_distraction"
  | "drink_water_first";
```

```ts
type FullnessCheck = {
  checkedAtSeconds: number;
  level: "30_percent" | "50_percent" | "70_percent" | "enough" | "too_much";
};
```

```ts
type MealResult =
  | "just_right"
  | "slightly_overate"
  | "overate";
```

```ts
type OvereatingCause =
  | "fast_eating"
  | "cooked_too_much"
  | "served_too_much"
  | "alcohol"
  | "distracted_eating"
  | "stress"
  | "mottainai"
  | "too_hungry"
  | "other";
```

```ts
type StrategyResult =
  | "success"
  | "partial"
  | "failed";
```

## 7. MVP要件

## 7.1 必須要件

- スマホ幅で自然に操作できる
- トップ画面から食事開始できる
- 今日の作戦を1つ選べる
- 食事タイマーが動く
- 1分ごとにリマインド文言が変わる
- 5分、10分で満腹度チェックを促す
- 満腹度を記録できる
- 食事終了後に振り返りを入力できる
- 記録をローカル保存できる
- 記録一覧を確認できる
- キャラの応援セリフが各画面に表示される
- ユーザーを責める表現を使わない

## 7.2 MVPの受け入れ条件

- 食事開始から記録保存まで一連の流れが完了できる
- ページを閉じても過去記録が残る
- 15分以上食べた場合に達成扱いになる
- 15分未満で終了しても記録は保存できる
- 満腹度チェックを複数回保存できる
- 記録画面に食事時間、作戦、結果、原因が表示される
- キャラ素材が未確定の場合でも、仮キャラ表示でUIが破綻しない
